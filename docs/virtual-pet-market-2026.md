# 電子雞產品研究（2026）

## 結論

Phubbing Anchor 最適合採用的產品組合是：

> **Focus Friend 的「現實行為換養成資源」＋ Widgetable 的「共同飼養」＋ Tamagotchi 的「分支進化」**

不要把即時 AI 聊天當成主軸。它會鼓勵使用者繼續看手機，與「聚會時減少 phubbing」的核心價值互相抵銷。AI 比較適合作為聚會結束後的回顧，例如根據本次專注時間與互動紀錄產生寵物日記，而不是陪聊。

## 建議的核心循環

1. 群組開始聚會並完成同步定錨。
2. 每人的有效專注分鐘轉成共同寵物的能量與養成素材。
3. 拿起手機或過早離場不直接處罰寵物，只會失去該段可獲得的資源，避免同儕責怪。
4. 聚會結束立即開獎：寵物成長、房間裝飾或進化進度。
5. 不同群組行為累積不同特質，例如「穩定專注」「常玩問答」「經常拍合照」，最後導向不同進化分支。

這個循環把現有的定錨、專注偵測、群組、聚會紀錄與共同寵物串成同一條產品敘事，不需要另造一個與主功能無關的小遊戲。

## 開發優先序

### P0：讓現有養成可信

- 狀態衰減必須穩定、可預測，不能因讀取 API 而重置。
- 多人同時餵食、玩耍、清潔必須使用資料庫交易。
- 聚會結束的專注獎勵必須與一般互動共用同一套狀態計算。

### P1：把專注成果變成可見成長

- 結算頁顯示「本次聚會替寵物獲得了什麼」。
- 加入一種可累積的進化進度或裝飾資源。
- 寵物頁顯示成長原因，而不只是 hunger／energy 數值。

### P2：增加長期記憶

- 依群組行為做 3 至 5 條進化路線。
- 聚會後生成短篇寵物日記；AI 只整理既有紀錄，不虛構事件。
- 將進化與聚會相簿、共同回憶結合，形成群組專屬資產。

## 口試定位句

> Phubbing Anchor 不是把電子雞搬到手機裡，而是把一群人真正放下手機、專心相處的時間，轉化成一隻共同寵物的成長歷程。
